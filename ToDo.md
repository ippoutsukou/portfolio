## 3) GitHubにportfolioサイト + 紹介アプリをアップロード
### 3-1. リポジトリ準備
- [x] GitHubにリポジトリ作成（例：`portfolio`）
- [x] サイト一式を同じ階層構造で配置  
  - [x] `index.html`
  - [x] `works.html`（存在する場合）
  - [x] `css/` `js/` `content/`（参照しているフォルダ）
- [x] READMEを用意（サイト概要、公開URL、連絡先）

### 3-2. 公開前セキュリティ（Codexチェック）
- [x] `AGENTS.md` をリポジトリ直下に配置
- [x] `.agents/skills/security-audit/SKILL.md` を配置
- [x] Codexで `security-audit` 実行（Secrets/不要ファイル/XSS/CDNチェック）
- [x] 指摘事項を修正（必要なら差分で反映）
- [x] 不要ファイル除去（backup/tmp/zip/logなど）
- [x] APIキー等がフロント（HTML/JS）に含まれていないことを再確認

### 3-3. GitHub Pages 公開
- [x] Settings → Pages → Deploy from a branch
- [x] Branch: `main` / folder: `(root)` を選択
- [x] 公開URLを確認（`https://<user>.github.io/<repo>/`）

### 3-4. 公開後の動作確認
- [ ] トップページ表示（崩れなし）
- [ ] リンク切れなし（works/コンテンツ/外部リンク）
- [ ] 外部CDN読み込み確認（フォント・アイコン・アニメーション等）
- [ ] スマホ表示確認（レイアウト崩れ／タップしやすさ）
- [ ] `content/` の公開物が意図通りか確認（見せたくない生データが無い）

### 3-5. アプリ紹介の整備
- [ ] アプリ紹介ページを整える（目的／機能／画面／使い方）
- [ ] デモ方法を記載（静的で動くなら「ブラウザで操作」）
- [ ] スクショに機密が無いことを確認
- [ ] ソース公開範囲を決める（同repo or 別repo）
- [ ] ライセンス（任意：MIT等）を検討
