## 1. プロジェクト概要

### 1.1 目的

製造業の新人社員が、自社の **商品・生産工場・仕向け先** の全体像と相互関係を短期間で理解できる、教育用の静的WEBアプリケーションを構築する。

### 1.2 背景と課題

| 項目 | 内容 |
|------|------|
| 商品数 | 約 40 種 |
| 生産工場数 | 約 10 拠点 |
| 仕向け先数 | 約 30 先 |

新人が業務で直面する問題は「覚える量が多いこと」ではなく、**何と何がどうつながっているか分からないこと** である。既存のExcelや紙資料は一覧性はあるものの、商品・工場・仕向け先の関係性をたどりづらく、現場での取り違えや選択ミスにつながっている。

### 1.3 プロダクト名（案）

- 新人教育ポータル
- 商品・工場・仕向け先ナビ
- 製品学習ガイド

### 1.4 ポジショニング

本アプリは「業務システム」ではなく **「新人が迷わないための業務ナビゲーション資料」** である。入力・編集機能は初版では設けず、教育コンテンツの閲覧と理解に特化する。

## 3. ユーザー要件

### 3.1 想定ユーザー

| ユーザー区分 | 用途 |
|-------------|------|
| 新人社員（主対象） | 商品・工場・仕向け先の学習、業務中の参照 |
| 教育担当者 | 教育の進行補助、コンテンツ更新（JSON編集） |
| 現場社員 | 日常業務での簡易リファレンス |

### 3.2 ユーザーストーリー

1. **新人として**、全体にどれだけの商品・工場・仕向け先があるか把握したい。
2. **新人として**、ある商品がどの工場で作られ、どこに出荷されるかを即座に調べたい。
3. **新人として**、名前が似ている商品の違いを理解して取り違えを防ぎたい。
4. **新人として**、仕向け先ごとの特有ルールや注意点を確認したい。
5. **新人として**、まず何から覚えればよいか迷わず学習を進めたい。
6. **新人として**、理解度を自分で確認できるクイズに取り組みたい。
7. **教育担当者として**、商品の追加・変更をJSONファイルの編集で反映したい。
8. **現場社員として**、業務中に商品コードや工場名からすぐに情報を引きたい。

## 5. 非機能要件

### 5.1 技術要件

| 項目 | 仕様 |
|------|------|
| アーキテクチャ | 静的WEBアプリケーション（SPA風構成） |
| フロントエンド | HTML / CSS / JavaScript（フレームワークは任意） |
| データ形式 | JSON ファイル |
| ホスティング | GitHub Pages / 社内ファイルサーバ / ローカル閲覧 |
| ビルドツール | 不要（またはViteなど軽量なもの） |
| サーバーサイド | なし |

### 5.2 ブラウザ対応

| ブラウザ | バージョン |
|----------|-----------|
| Google Chrome | 最新2バージョン |
| Microsoft Edge | 最新2バージョン |
| その他 | 基本対応（レイアウト崩れは許容） |

### 5.3 レスポンシブ対応

| デバイス | 対応方針 |
|----------|---------|
| デスクトップ（主対象） | フル機能 |
| タブレット | 基本機能。レイアウトは自動調整 |
| スマートフォン | 閲覧可能。一部レイアウト簡略化 |

### 5.4 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| 初回ロード | 3秒以内（ローカル環境） |
| 画面遷移 | 即時（SPA内遷移） |
| 検索レスポンス | 入力から200ms以内にフィルタ反映 |

### 5.5 保守性

- HTMLは見た目（表示）、JSONはデータ（中身）で明確に分離する。
- 商品の追加・変更はJSONファイルの編集のみで反映可能とする。
- コードにはコメントを適切に記載し、教育担当者がデータ更新できるレベルを目指す。

### 5.6 アクセシビリティ

- セマンティックHTMLを使用する。
- キーボード操作で基本的なナビゲーションが可能であること。
- 色のみに依存しない情報伝達を行う。

## 7. UI設計方針

### 7.1 基本原則

1. **一画面に詰め込みすぎない** — 分類して段階的に表示する。40商品を1ページに全表示しない。
2. **関係性で見せる** — 一覧を辞書で終わらせず、相互リンクで「つながり」を理解させる。
3. **現場の注意点を入れる** — マスタ情報だけでなく、間違えやすいポイントを必ず併記する。
4. **迷わせない** — 「全部載っている」より「どこを見ればいいか分かる」を優先する。

### 7.2 ナビゲーション構造

```
トップ
├── 商品一覧 → 商品詳細
├── 工場一覧 → 工場詳細
├── 仕向け先一覧 → 仕向け先詳細
├── 関係マップ
├── よくある間違い
├── 学習モード（クイズ含む）
└── 用語集 / FAQ
```

### 7.3 共通UIパターン

| パターン | 用途 |
|---------|------|
| カード一覧 | 商品・工場・仕向け先のブラウジング |
| テーブル一覧 | 詳細比較が必要な場面の切替表示 |
| インスタント検索 | 入力に応じたリアルタイムフィルタ |
| タグ絞り込み | カテゴリ・タグによるフィルタ |
| 相互リンクバッジ | 詳細画面から関連エンティティへの遷移 |
| アコーディオン | FAQ・用語集の展開表示 |
| ステッパー | 学習モードの進捗表示 |

### 7.4 カラー・トーン方針

- 業務資料として落ち着いた配色とする（原色は使わない）。
- カテゴリごとに識別しやすいアクセントカラーを割り当てる。
- 注意喚起には警告色（黄・赤系）を控えめに使用する。

## 10. 制約事項・前提条件

### 10.1 制約

- データ更新はJSONファイルの手動編集による。UI上の編集機能は初版に含まない。
- 複数人同時編集の仕組みは対象外。Gitによるバージョン管理で運用する。
- 承認フローは本アプリの範囲外。コンテンツの正確性は教育担当者が確認する。

### 10.2 前提条件

- 商品・工場・仕向け先のマスタ情報がExcelまたは紙で存在する。
- 現場の注意点・よくあるミスの情報は教育担当者またはベテラン社員からヒアリングで収集する。
- 利用者のPC環境にChrome or Edgeがインストールされている。

## 12. 受け入れ基準

本アプリの初版リリースにあたり、以下を満たすこと。

1. 全商品（40種）の基本情報が閲覧可能であること。
